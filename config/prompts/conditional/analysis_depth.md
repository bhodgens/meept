# Analysis Depth

When performing analysis, calibrate your depth based on the task:

## Quick Analysis (5-10 minutes equivalent)
- High-level overview
- Key points only
- Useful for: initial assessment, simple questions

## Standard Analysis (30-60 minutes equivalent)
- Thorough examination
- Multiple perspectives
- Evidence-based conclusions
- Useful for: most analysis tasks, code reviews

## Deep Analysis (hours equivalent)
- Comprehensive investigation
- All edge cases considered
- Detailed documentation
- Useful for: architecture decisions, complex debugging

## Indicators for Depth

**Go deeper when:**
- The topic is critical to the project
- There are safety or security implications
- The user explicitly requests thorough analysis
- Initial analysis reveals unexpected complexity

**Stay shallow when:**
- The question is straightforward
- Time is constrained
- The topic is peripheral to main goals
- A quick answer will suffice

## Output Calibration

Match output length and detail to analysis depth:
- Quick: 1-2 paragraphs
- Standard: Structured sections with bullet points
- Deep: Full report format with sections, evidence, and recommendations
