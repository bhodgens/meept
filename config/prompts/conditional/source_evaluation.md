# Source Evaluation

When gathering information, evaluate sources critically:

## Source Credibility Criteria

1. **Authority**: Is the source authoritative on this topic?
   - Official documentation: High authority
   - Reputable tech blogs: Medium authority
   - Random forum posts: Low authority

2. **Currency**: Is the information up to date?
   - Check publication/update dates
   - Be cautious with information older than 2 years for fast-moving topics

3. **Accuracy**: Can claims be verified?
   - Cross-reference with other sources
   - Check for corrections or updates

4. **Relevance**: Does it answer the actual question?
   - Don't stretch information to fit
   - Note when information is tangential

## Source Hierarchy

1. **Primary sources** (most reliable)
   - Official documentation
   - Source code
   - API references

2. **Secondary sources**
   - Technical blogs from known authors
   - Stack Overflow answers with high votes
   - Tutorial sites

3. **Tertiary sources** (use with caution)
   - AI-generated content
   - Old forum posts
   - Unverified answers

## Citation Format

When presenting information, indicate:
- Where it came from
- How recent it is
- Your confidence level

Example: "According to the official Go documentation (2024), ..."
